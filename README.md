# defense-ppt-outline-writer

`defense-ppt-outline-writer` 是一个用于学术论文答辩与学术展示准备的 Codex skill。它可以读取完整的论文、开题报告、研究计划、课程论文、会议论文或项目报告等文档，按照原文结构拆解内容，精简为适合 PPT 展示的页级大纲，并进一步生成对应的讲稿。

## 适用场景

适合在以下任务中使用：

- 将 `.docx`、PDF、Markdown 等论文/开题报告转成答辩 PPT 内容大纲
- 将论文、研究报告、课程作业、会议汇报材料转成学术展示 PPT 内容大纲
- 从完整文档中提炼研究背景、研究意义、文献综述、研究缺口、理论基础、研究内容、研究问题、研究方法、论文框架、研究发现、讨论与结论等内容
- 在不大幅改写原文的前提下，保留重点、压缩篇幅
- 将“大点—小点—段落说明”的层级关系整理清楚
- 基于 PPT 大纲继续写一版口头答辩稿、展示稿或汇报稿

## 核心能力

该 skill 的默认工作方式是：

1. 读取完整源文档
2. 识别文档标题、章节结构、正文段落、目录条目和参考文献
3. 按答辩/展示逻辑重新组织内容
4. 精简正文，但尽量保留原文原句、原文短语和原文目录条目
5. 输出可直接用于制作 PPT 的 Markdown 大纲
6. 基于大纲生成逐页讲稿

## 调用方式

```text
调用 defense-ppt-outline-writer，读取 /path/to/论文或报告.docx，帮我拆成适合学术答辩/展示 PPT 的内容大纲，并写一版讲稿。
```

更具体的调用方式：

```text
调用 defense-ppt-outline-writer，读取这个论文/研究报告。
要求：
1. 按结构拆解，精简内容保留重点；
2. 大点下面拆小点，不要全部平铺；
3. 正文尽量保留原文原句、原文短语和原文目录条目；
4. 输出可以直接做 PPT 的内容大纲；
5. 再基于这个大纲写一版讲稿。
```

## 推荐输入

推荐提供：

- 源文档路径，例如 `.docx`、`.pdf`、`.md`
- 展示类型，例如开题答辩、预答辩、毕业答辩、课程汇报、会议汇报、论文展示、项目汇报
- 期望页数或时长，例如 15-20 页、10 分钟、15 分钟
- 是否需要保留参考文献
- 是否需要更精简或更完整

示例：

```text
调用 defense-ppt-outline-writer，读取 /Users/me/project/论文.docx。
做成 18 页左右的学术展示 PPT 大纲，保留参考文献，正文尽量用原句。
讲稿按 10 分钟展示来写。
```

## 默认输出

默认会生成两个 Markdown 文件：

```text
<源文件名>_PPT内容划分_层级段落版.md
<源文件名>_讲稿_层级段落版.md
```

第一个文件是 PPT 内容大纲，通常包含：

- 第几页
- 页面标题
- `PPT正文：`
- 页面正文
- 一级点
- 二级展开
- 必要的收束句或核心问题

第二个文件是讲稿，通常按 PPT 页码逐页展开，帮助讲清楚研究选题、问题意识、研究设计、研究发现和预期贡献。

## 输出结构示例

```markdown
## 第7页：研究缺口

PPT正文：

从本文的问题意识出发，现有研究仍有进一步推进空间。

- 单一平台研究较多，跨平台考察不足。
  - 既有研究较多聚焦单一平台，微信和小红书常常被分别研究，缺少二者并置情境下的跨平台考察。

- 公开表达研究较多，动态关系过程关注不足。
  - 现有研究较多关注公开表达、内容发布和消费互动，对聊天、私信、评论转移、分享给谁、是否加微信等动态关系过程关注不足。
```

对应讲稿/展示稿示例：

```markdown
## 第7页：研究缺口

这一页主要说明本文的问题意识。也就是说，虽然既有研究已经为本文提供了重要基础，但从本文关注的跨平台聊天场景来看，仍然存在进一步推进的空间。

首先，现有研究中单一平台研究较多，跨平台考察不足……
```

## 内容处理原则

该 skill 默认遵循以下原则：

- 保留重点，而不是完整复制全文
- 保留原文表达，而不是随意改写
- 允许合并页面，但不打乱核心论证逻辑
- 用段落承接复杂论述，用分点呈现并列结构
- 大点下面拆小点，避免所有内容平行堆叠
- 讲稿可以加入自然过渡，但不新增未经来源支持的观点

## 推荐 PPT 结构

对于论文开题、预答辩、正式答辩等研究设计型展示，默认推荐如下结构：

1. 封面
2. 研究背景
3. 核心研究问题
4. 研究意义
5. 国内外研究述评
6. 研究缺口
7. 理论基础与核心概念
8. 研究内容
9. 研究问题
10. 研究思路
11. 研究方法
12. 论文框架
13. 研究创新点
14. 研究难点与可行性
15. 研究计划
16. 参考文献

对于论文成果展示、会议汇报或课程论文展示，可以调整为：

1. 封面
2. 研究背景与问题意识
3. 文献基础与研究缺口
4. 理论框架/分析框架
5. 研究设计与方法
6. 材料、样本或案例
7. 核心发现/分析结果
8. 讨论
9. 贡献与不足
10. 结论
11. 参考文献

实际页数会根据源文档内容和用户要求调整。

## 注意事项

- 如果用户要求“全部原句”，输出会更长；如果要求“适合 PPT”，需要适度压缩。
- 页面标题和结构标签可能是为组织答辩逻辑而生成的，不一定来自原文。
- PPT 正文会尽量使用原文原句、原文短语和原文目录条目。
- 讲稿不是逐字朗读稿，而是面向评审老师、课堂听众或会议听众的解释性表达。
- 如果后续还要制作 PPT，可以继续把输出的大纲交给 PPT 生成类 skill 使用。
